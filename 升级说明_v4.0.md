## 🐛 踩过的坑与经验教训

### 1. 图标显示问题
**问题**：任务栏显示字母"e"，文件夹显示 3.2 版本图标
**原因**：
- 资源文件路径配置错误
- 图标文件未正确复制到 `src` 目录
- Windows 缩略图缓存问题

**解决方案**：
- 将 4.0 版 `icon.ico` 复制到 `src` 目录
- 更新 `resources.rc` 指向正确路径
- 修改 `main.rs` 中的 `load_icon` 函数使用 `icon.jpg`
- 用户端刷新缩略图或重命名可执行文件

**经验**：Windows 资源编译要确保路径绝对正确，图标文件要放在正确位置

---

### 2. 命令行模式和 AI 处理无输出
**问题**：GUI 能正常工作，但 CLI 和 JSON 模式无输出
**原因**：
- `main` 函数未检查命令行参数，直接启动 GUI
- 参数解析逻辑缺失

**解决方案**：
- 修改 `main` 函数添加命令行参数检查
- 区分 GUI 和 CLI 模式
- 实现 `run_cli` 和 `run_json_mode` 函数

**经验**：多入口程序要确保所有入口点都能正常工作

---

### 3. 右键发送到功能无效
**问题**：多选文件通过右键发送到程序无反应
**原因**：
- `Cli` 结构体仅支持显式 `--input` 参数
- 未处理位置参数

**解决方案**：
- 在 `Cli` 结构体中添加 `positional` 字段
- 修改 `run_cli` 函数同时处理显式参数和位置参数

**经验**：Windows SendTo 是通过位置参数传递的，要同时支持两种参数方式

---

### 4. 配置文件位置问题
**问题**：处理文件时 `image_compressor_config.toml` 出现在桌面
**原因**：
- 配置文件默认保存路径为当前目录
- 从桌面启动程序时当前目录就是桌面

**解决方案**：
- 使用 `dirs` 库获取系统配置目录
- 将配置文件保存到 `%APPDATA%\rust_image_compressor\config.toml`

**经验**：配置文件应该放在系统标准位置，不要污染用户目录

---

### 5. GUI 界面方框乱码
**问题**：界面上出现方框，无法正常显示
**原因**：
- 使用了 emoji 表情
- 某些系统环境对 emoji 支持不佳

**解决方案**：
- 移除所有 emoji 表情
- 使用纯文本标签
- 保持与 3.2 版本一致的界面风格

**经验**：Windows GUI 要避免使用 emoji，兼容性问题太多

---

### 6. PNG 压缩效果不佳
**问题**：PNG 图片处理后大小几乎没变
**原因**：
- 使用 `image` 库的默认设置
- 未启用压缩优化参数

**解决方案**：
- 使用 `PngEncoder::new_with_quality`
- 设置压缩级别为 `CompressionType::Best`
- 设置过滤方法为 `FilterType::Adaptive`

**经验**：PNG 压缩一定要显式设置压缩参数，默认设置几乎不压缩
